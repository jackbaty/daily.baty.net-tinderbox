Markdown in Tinderbox

The rich text editor in Tinderbox works fine, and does automatic conversions to HTML during exports. This is great, but I am more comfortable writing posts using Markdown. Tinderbox supports Markdown, but when exporting, it will need to call the markdown processor for every Markdown note. I am concerned about the performance hit over time. It already takes 10 seconds to render the blog without Markdown. I know this doesn’t seem like much, but I’m used to Hugo, which rendered my 3,000-post blog in under three seconds. I suppose I could try it and see.
